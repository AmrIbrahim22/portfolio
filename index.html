The full picture — not just the code

Most job descriptions say "Flutter developer." Here's what I actually show up and do every day.

01 🏗️

Architecture & Flutter Code

Clean Architecture, BLoC/Cubit, Repository pattern, Drift ORM, GoRouter. Production-grade Dart that the next developer can confidently read, extend, and not be afraid of.

02 🔗

Backend Coordination

I debug with the backend team — not against them. API contracts, error handling, Azure App Service issues, response schema changes. When the server goes down I know what to tell them and why.

03 🏪

Store Submission & Launch

App Store & Play Store from scratch — provisioning profiles, metadata, screenshots, review responses, rejection handling. I know what they'll reject before I submit.

04 🔄

Azure DevOps & CI/CD

Azure Pipelines, work items, Git branching strategy, automated builds and deployments. If your team manually builds and uploads every release, I'll automate that in the first sprint.

05 🧭

Scrum & Team Leadership

Running standups, sprint planning, stakeholder alignment. I translate between business goals and engineering reality — including when to say "that feature takes 3 days, not 3 hours."

06 📈

Market Fit & Maintenance

Shipping isn't the finish line. I monitor live apps, push OS-compatibility updates, watch crash reports, and iterate based on what real users actually do — not what was assumed in planning.
